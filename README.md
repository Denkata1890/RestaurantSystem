Този проект представлява цялостна система за управление на ресторант, включваща RESTful API (Back-end) и клиентска част (Front-end). Системата позволява управление на менюта, служители, маси и поръчки чрез CRUD операции.

Основни функционалности:

    Сигурност: Всички услуги са защитени с JWT (JSON Web Token).
    База данни: 6 свързани таблици със сложни релации.
    Търсене: Реализирано филтриране по специфични критерии за всяка таблица.
    Странициране: Вградена логика за извличане на списъци с данни на части 
    Документация: Автоматично генерирана Swagger (Open API) документация

Технологичен стак:
    Език: C#
    Платформа: .NET 8.0 / .NET Core
    ORM: Entity Framework Core (Code First модел)
    База данни: SQL Server 2019
    Автентикация: JWT Bearer Token


Структура на базата данни

    Categories – Категории храни 
    MenuItems – Ястия в менюто
    Tables – Маси в ресторанта
    Employees – Персонал
    Orders – Поръчки 
    OrderDetails – Детайли на поръчка







    Инструкции за инсталация и стартиране
1. Клониране на репозиторието
Bash
git clone https://github.com/[твоят-username]/RestaurantSystem.git
cd RestaurantSystem
2. Настройка на базата данни
Отворете appsettings.json.

Променете ConnectionStrings:DefaultConnection, за да сочи към вашия локален SQL Server.
Пример: Server=.;Database=RestaurantDb;Trusted_Connection=True;

Отворете Package Manager Console във Visual Studio и изпълнете:

PowerShell
Update-Database
3. Стартиране на API-то
Натиснете F5 във Visual Studio.

Ще се отвори Swagger интерфейс на адрес https://localhost:[port]/swagger.

4. Тестване на защитените услуги
Използвайте POST /api/Auth/login с потребител admin и парола password123.

Копирайте върнатия токен.

Натиснете бутона Authorize в Swagger и въведете: Bearer [вашият-токен].

📂 Допълнителни характеристики
Валидация на входящите данни чрез Data Annotations.

Използване на различни типове данни за покриване на критериите.

Връзки тип Едно-към-Много и Много-към-Много (през OrderDetails).
